# ADR-0002：法球用 FIFO 队列建模

- 状态：Accepted
- 日期：2026-07-24

## 背景

Dota2 卡尔的法球机制：头顶至多 3 个元素球，切新球时旧球按先进先出顺序被挤出。
球序（而不仅是球的集合）会影响后续技能的最少切球数——例如头顶 [Q,W] 时切 E
得到 [Q,W,E]；而头顶 [W,E] 时切 Q 得到 [W,E,Q]，两者都能合成同一技能但起手状态不同。

## 决策

**法球用 FIFO 队列建模**，而非无序集合：

- `Orb` 是 `Element[]` 数组，队首在下标 0，队尾在末尾
- 切球：队列满（3）时 `orbs = [...orbs.slice(1), newKey]`（队首出队，新球入队尾）
- 合成判定用**多重集匹配**（顺序无关），但**球序在合成后完整保留**，直接影响下一技能

这一建模是 `solver`（最优按键求解器）的核心前提：solver 在 `(orbs × progress × slots)`
状态空间里用 0-1 BFS 求精确最少切球数，必须追踪球的真实顺序。

## 结果

- solver 能求出精确最优解（比早期多重集贪心更准：贪心既会高估也会低估）
- 预切起手（PreCast）机制得以正确实现：起手球序 = 最后合成的预切技能配方
- FIFO 队列递进动画（framer-motion）有了语义基础：新球入队尾、队首被推出
