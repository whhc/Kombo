# 01 — 项目骨架 + 单元素球可视化(端到端打通)

Status: ready-for-agent

## Parent

无(本切片为 v1-mvp 的根切片,doc.md v1.1 为方案来源)。

## What to build

搭建 Kombo 的可运行骨架,并实现第一条端到端链路:玩家在 Kombo 窗口内按 `Q`/`W`/`E`,UI 实时显示卡尔头顶元素球队列的更新。

具体行为:

- 初始化 Tauri v2 + React + TypeScript + Tailwind CSS 项目骨架,产物可本地 `dev` 启动、可打包。
- 在窗口(或 window)级别注册 DOM `keydown` 监听,捕获玩家在窗口内的按键(不使用 rdev 全局钩子,见 doc.md §0/§1)。
- 过滤层:仅保留 `Q`/`W`/`E`,其余键忽略;每个合法按键附带 `performance.now()` 毫秒时间戳。
- 状态机切球逻辑:维护头顶最多 3 个元素球的队列,先进先出(FIFO)—— 生成第 4 个元素时,最早生成的第 1 个被挤出(doc.md §2.1)。
- 元素球 UI:实时渲染当前 3 个球(Q=冰/W=雷/E=火,可先用颜色区分),按键后立即更新。

此切片是 tracer bullet —— 它一次性压测了"空仓库→可跑应用 + 键盘监听链路通 + 状态机 + UI 渲染"四件事,后续所有切片都在此骨架上增量。

## Acceptance criteria

- [ ] `pnpm tauri dev`(或等价命令)能成功启动一个桌面窗口,显示元素球区域。
- [ ] 在窗口内按 `Q`,UI 上出现一个"冰(Q)"球;按 `W`、`E` 同理出现"雷"、"火"球。
- [ ] 球按 FIFO 进出队:按满 3 个球后,按第 4 个键时最早那个球被挤出,新球进入队尾,UI 正确反映。
- [ ] 按非 `Q`/`W`/`E` 的键(如 `A`、`空格`)不产生任何状态变化或 UI 抖动。
- [ ] 每个合法按键被记录时附带毫秒时间戳(供后续切片使用,本切片 UI 可不展示)。
- [ ] 切球逻辑与 UI 渲染有单元测试覆盖(至少:FIFO 进出队、忽略非法键)。
