# 02 — 祈唤与槽位状态机

Status: ready-for-agent

## What to build

在切球状态机之上实现祈唤(Invoke)与双技能槽位状态机,严格遵循 doc.md §2.2。

具体行为:

- 玩家按 `R` 时,提取当前头顶 3 个元素,**忽略排列顺序**合成判定(QWE 与 EWQ 视为相同),映射到 §2.3 技能字典里的对应技能(或无对应 = 无效祈唤)。
- 双槽位状态机:维护"第一顺位"与"第二顺位"两个槽位,按 §2.2 三规则更新:
  - 新切出的技能**不在**当前槽位中 → 新技能进第一顺位,原第一顺位被推至第二顺位,原第二顺位被移除。
  - 新切出的技能**已在第一顺位** → 槽位顺序不变。
  - 新切出的技能**在第二顺位** → 该技能提至第一顺位,原第一顺位降至第二顺位。
- 槽位 UI:实时渲染两个槽位当前挂着的技能(图标 + 名称),祈唤后立即反映状态变化。
- 无效祈唤(3 球组合不在 10 个技能表内,如 QQQ 是合法 ColdSnap、但若扩展时出现无映射组合)按无效处理,不改槽位。

此切片延续 tracer bullet —— 把状态机从"切球"扩展到"祈唤+槽位",为释放做准备。

## Acceptance criteria

- [ ] 按 `Q` `Q` `Q` `R` → ColdSnap 进入第一顺位,UI 显示第一槽 = ColdSnap、第二槽 = 空。
- [ ] 在槽位非空时祈唤出**不在槽位**的新技能 → 新技能进第一顺位,原第一顺位降至第二顺位,原第二顺位被移除。
- [ ] 祈唤出**已在第一顺位**的技能 → 两个槽位顺序不变(本切片不模拟冷却,§2.5 已约定)。
- [ ] 祈唤出**在第二顺位**的技能 → 该技能提至第一顺位,原第一顺位降至第二顺位。
- [ ] 合成判定忽略元素排列:切 `Q` `W` `E` 与切 `E` `W` `Q` 后按 `R`,均切出 DeafeningBlast。
- [ ] §2.2 三规则的槽位状态转移有单元测试覆盖(建议以 reducer/状态机表驱动测试,便于后续评估器复用)。
