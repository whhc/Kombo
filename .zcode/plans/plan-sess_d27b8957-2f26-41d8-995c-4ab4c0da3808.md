## i18n + 图标接入 实施计划

### 决策汇总(已与用户对齐)
- **i18n**:轻量自建,零依赖(translations 字典 + `useT()` hook + localStorage 记语言)
- **图标展示**:**仅图标,名称作 tooltip**(中文 tooltip 显中文名,英文显英文名)
- **语言切换**:显式按钮(与 SettingsBar 同区域),localStorage 记忆
- **默认语言**:中文

### 实施步骤(垂直切片 TDD,每步独立提交)

#### Step 1 — i18n 基础设施 + 图标 SSOT
新增文件(纯逻辑,无 React,先 TDD):
- `src/domain/icons.ts`:SpellName → 图标文件名映射 SSOT(处理 `EMP → E.M.P._icon.webp` 等不规则名),导出 `spellIconUrl(spell)` / `elementIconUrl(element)`。用 `import.meta.glob` 或静态 import 加载 webp(Vite 资产处理)。
- `src/domain/i18n.ts`:`Locale = 'zh' | 'en'` 类型;`translations: Record<Locale, Dict>` 字典,含所有 UI 文案 + 技能/元素名(`spellNames`、`elementNames` 嵌入字典,淘汰现有 `spellNames.ts` 的 `SPELL_CN`)。`t(locale, key)` 纯函数。
- `src/hooks/useLocale.ts`:`useLocale()` hook 返回 `{ locale, setLocale, t }`,localStorage 记忆。
- 测试:`i18n.test.ts`(zh/en key 都存在、缺失 key 回退到 key 本身或 zh)、`icons.test.ts`(映射完整性)。

#### Step 2 — 预设连招 name 改 i18n key
- `presets.ts`:预设的 `name` 改为 i18n key(如 `preset.tornadoEmpMeteorBlast`)。
- 数据模型 `TargetCombo.name` 语义微调:用户自建 = 原样文本;预设 = i18n key。加注释说明。
- UI 展示连招名时,若 name 以 `preset.` 开头则 `t(name)` 翻译,否则原样显示。封装一个 `resolveComboName(combo, t)` 纯函数(放 i18n.ts 或 combo.ts)。
- 旧数据兼容:localStorage 里已存的预设仍带中文名 —— 无害(首次加载 seed 时按 comboId 覆盖更新)。

#### Step 3 — SpellIcon / ElementIcon 组件 + 替换现有展示
- `src/components/SpellIcon.tsx`:`<SpellIcon spell tooltip? size? />` —— 渲染图标 webp,`title` 属性做 tooltip(原生 HTML tooltip,零依赖),tooltip 文案走 i18n 技能名。
- `src/components/ElementIcon.tsx`:同理渲染 Q/W/E 元素图标。
- 重构 `OrbDisplay`:用 ElementIcon 替换当前色块,保留 aria-label(走 i18n)。
- 重构 `SlotDisplay`:用 SpellIcon 替换当前文字,tooltip=i18n 技能名。
- 重构 `ProgressBar`:技能用 SpellIcon + 小字序号。
- 重构 `ComboManager` / `ComboEditor`:技能选项/序列用 SpellIcon(编辑器的"添加 强袭飓风"按钮改成图标+tooltip)。

#### Step 4 — UI 文案 i18n 化(App + 所有组件)
- 把所有组件 + App.tsx 的硬编码中文字符串改为 `t('key')` 调用。
- 文案 key 分组:`nav.*`(练习/连招库/复盘)、`common.*`(保存/取消/删除/编辑)、`practice.*`(当前连招/释放/空放/结束并保存/再练一次/引导)、`combo.*`(连招库/新建连招/预切槽/技能序列)、`dashboard.*`(时间范围/今日/近7天/近30天/空状态/节奏散点图)、`settings.*`(图标/键位)、`metrics.*`(切球达成率/多切次数/时长)。
- 新增 `SettingsBar` 里的语言切换按钮(中/EN)。
- 测试调整:组件测试里的中文字符串匹配改为用 i18n key 或保留中文(因为默认 zh,测试默认渲染中文仍可通过)。新增一个测试:切到 en 后关键文案变英文。

#### Step 5 — doc.md 同步 + 收尾
- doc.md §7 实现期决策追加 i18n + 图标接入的记录。
- 跑全套测试 + tsc + build 确认。

### 关键技术点
1. **图标加载**:用 Vite 的静态 import(`import tornadoIcon from '../icons/dota2/Tornado_icon.webp'`)或 `new URL('...', import.meta.url)`。因 dota1 图标待补,映射表先用 dota2 路径,后续 `iconTheme` 切换时按 theme 选目录(dota1 目录暂 fallback 到 dota2)。
2. **tooltip**:用原生 HTML `title` 属性,零依赖,符合"简单优先"。
3. **i18n key 缺失策略**:`t()` 找不到 key 时回退到 key 本身并 console.warn(开发期发现遗漏),生产可关。
4. **测试策略**:默认 zh,组件测试大部分无需改(仍匹配中文);新增少量 en 切换测试覆盖 i18n 真的生效。

### 不做的事(明确边界)
- 不引入 i18n 库(react-i18next 等)。
- 不做复数/性别/时态等高级 i18n 特性(中英两语言用不上)。
- 不改 domain 层纯逻辑(切球/评估等不涉及 UI 文案,无需 i18n)。
- 不做 dota1 图标(资源未提供,fallback dota2)。
- 不做 SSR/懒加载语言包(单页应用无需)。

### 风险与缓解
- **文案遗漏**:i18n key 多,可能漏。缓解:`t()` 缺失 key 时 warn,跑一遍应用自查;key 集中在 i18n.ts 一个文件便于盘点。
- **图标映射笔误**:`EMP → E.M.P._icon.webp` 这种特殊名易错。缓解:`icons.test.ts` 校验每个 SpellName 都有对应文件存在(用 `import.meta.glob` 列出实际文件比对)。
- **localStorage 旧数据**:已存预设连招的 name 是中文。缓解:seed 时按 comboId 覆盖(现有 saveCombo 逻辑已支持)。
